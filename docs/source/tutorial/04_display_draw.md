屏幕绘制
==============

## 分辨率与二维数组

前文[模拟器架构](03_emulator.md)介绍了 Chip8 的主循环中，当执行完 CPU 和键盘事件之后，需要绘制屏幕。将程序执行结果在显示器上显示。

Display 模块专门用来绘制图形。Chip8 的分辨率是`64 * 32`。这个分辨率在现代的机器上太小了。因此可以放大显示，设置宽 640，高 320 的窗口。一个像素就是一个 `10 * 10`的矩形块。

这样屏幕上一共有 64 * 32 个像素，可以使用一个二维数组进行表示，就是`List[List[int]]`的结构，一维数组是 y 轴的坐标，二维是 x 轴坐标，x 轴的每个元素代表一个像素。[^1] 

每个像素的值是**0**或者**1**，0 就显示背景色，1 就显示前景色。绘制图形的时候，遍历这个二维数组，根据 x，y 轴的坐标和数组中的值绘制矩形，最终显示出图像。

## 实现

了解屏幕显示与二维数组的关系之后，剩下的就是解析二维数组调用 pygame 进行绘图。

### 绘制一像素的方法：

使用 pygame 绘制一个矩形。

```{eval-rst}
.. automethod:: src.tutorial.04.1_display.Display.draw_pixel
```

```python 
class Display:

    def draw_pixel(self, x: int, y: int):
        pygame.draw.rect(
            self.canvas,
            FG_COLOR,
            (x * ZOOM, y * ZOOM, ZOOM, ZOOM),
            0
        )
```

### 绘制一帧的方法:

```{eval-rst}
.. automethod:: src.tutorial.04.1_display.Display.draw_frame
```

```python 
    def draw_frame(self, screen_buf: List[List[int]]):
        self.canvas.fill(BG_COLOR) # 先清屏
        for y in range(SCREEN_HEIGHT):
            for x in range(SCREEN_WIDTH):
                if screen_buf[y][x] == 1:
                    self.draw_pixel(x, y)
```

遍历二维数组 screen_buf，坐标值定位的二维数组元素值为 1，则绘制一个像素。把所有像素都绘制完毕，再渲染屏幕。需要注意的时候，每次绘制一帧的时候，需要先清屏。

### 渲染canvas的方法:

    def render(self, screen_buf: List[List[int]]):
        self.draw_frame(screen_buf)
        pygame.display.update()

## 总结

Chip8 的屏幕分辨率是`64 * 32`，可以映射为一个二维数组----`[32][64]uint8`。二维数组的每一个元素，即屏幕上的一个像素，为一个 `10 * 10` 的矩形块。

二维数组元素的值是 `0`，则绘制背景色，为`1`则绘制前景色。通过像素块的前后背景色显示图像。

每绘制一帧，需要遍历一遍二维数组。绘制每一帧之前，先清屏。绘制完一帧之后，再刷新屏幕。

接下来，我们将使用 Chip8 解析 rom 文件，通过解析 CPU 指令执行够着二维数组，然后调用 Display 的 render 方法展示真正的 IBM 图像。

[^1]: Chip8的分辨率是 64 * 32，可以设置一个 比例 Zoom。相应的扩展 宽和高以及像素点即可。例如 Zoom=10，那么分辨率是 （64 * Zoom, 32 * Zoom)。像素点就是 1 * Zoom 的矩形块。
